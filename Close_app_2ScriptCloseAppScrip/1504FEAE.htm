The test run has stopped because --

  • An error message was posted to the log and
  • The On error project property is set to "Stop project" or "Stop test item".

If you don't want the test to stop on errors, set this property (Project Editor ⇒ Properties ⇒ Playback), to "Continue running" or change your test commands so that they do not post errors to the log automatically. For example, if a window or process was not found, use WaitWindow, WaitProcess or similar methods to get the needed object.
